Task C
Coluna Intent
+ Coaching Email

Score de 1 a 5 que representa o quanto o prospect demonstrou intenção de compra durante a call — calculado pela IA com base na transcrição. A IA não infere intent a partir do resultado: ela lê o que o prospect disse.

Médio Sem dependências
Regras de cálculo
result === 'closed' → intent é sempre 5. Regra fixa, sem análise de transcrição.
result !== 'closed' → intent calculado pela IA: tom do prospect, perguntas, objeções, engajamento, sinais de interesse. Não segue o resultado mecanicamente — um partial pode ter intent 2, um no_outcome pode ter intent 4.

Task D
Script Gap
Detection

A IA analisa 3 calls da organização e cruza duas perspectivas: o que o script instrui o vendedor a fazer, e o que está acontecendo na conversa na prática — tanto do lado do vendedor quanto do prospect. O resultado é uma sugestão cirúrgica de reescrita apenas do trecho com atrito.

Grande Sem dependências Somente Owner
Diferença da feature existente
Script Intelligence /dashboard/insights — analisa qualidade e cobertura do script do ponto de vista do vendedor. Ex.: "A seção de Objection Handling está fraca — o script não cobre a objeção de preço."
Script Gap Detection /script-gap — analisa a conversa como um todo (vendedor e prospect) e detecta onde o script gera atrito na prática. A sugestão é baseada tanto no comportamento do vendedor quanto nas reações, objeções e padrões de linguagem que o prospect está usando naquelas calls. Ex.: "O script instrui o vendedor a apresentar o programa completo logo após a descoberta, mas os prospects dessa org interrompem com objeções de agenda antes de ouvir a oferta — e o vendedor não tem no script uma resposta para isso. Sugestão: mover a apresentação para depois de tratar a objeção de tempo."
Estrutura de tipos
type ScriptGap = {
  id: string
  section: string          // ex: "Objection Handling"
  scriptInstruction: string // o que o script instrui o vendedor a fazer
  observedPattern: string   // o que a IA detectou na conversa — vendedor e prospect
  frequency: number       // % das calls analisadas onde o padrão aparece
  severity: 'high' | 'medium' | 'low'
  suggestedFix: string     // nova redação apenas do trecho com atrito
}

type ScriptGapAnalysis = {
  analyzedAt: string
  callsAnalyzed: string[] // 3 call IDs
  gaps: ScriptGap[]
}

Task F
Script Builder
Seções Fixas

O Script Builder atual é um wizard de 4 etapas que permite adicionar, remover e desativar seções livremente. A nova regra é que as seções do script são sempre fixas e imutáveis — as mesmas 5 seções em toda e qualquer criação de script, sem exceção. O usuário pode editar apenas o nome do script, a descrição, o peso de cada seção e se uma seção é crítica.
